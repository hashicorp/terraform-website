# API Changelog

Keep track of changes to the API for Terraform Cloud and Terraform Enterprise.

### 2022-05-11

* Updated Run Tasks permission to the following endpoints:
  * [Organizations](/cloud-docs/api-docs/organizations#list-organizations).
  * [Team Access](/cloud-docs/api-docs/team-access#list-team-access-to-a-workspace).
  * [Teams](/cloud-docs/api-docs/teams#list-teams).

### 2022-05-04

* Updated [Feature Sets](https://www.terraform.io/cloud-docs/api-docs/feature-sets#list-feature-sets) to add new `run-tasks` attribute.

### 2022-05-03

* Added Run Tasks permission to the following endpoints:
  * [Organizations](/cloud-docs/api-docs/organizations#list-organizations)
  * [Workspaces](/cloud-docs/api-docs/workspaces#show-workspace)

### 2022-04-29

Updated [Run Tasks Integration](/cloud-docs/api-docs/run-tasks-integration.mdx) to specify the allowed `status` attribute values.

### 2022-04-04

* Added the [Run Tasks Integration](/cloud-docs/api-docs/run-tasks-integration) endpoint.

### 2022-03-14

* Added the [Download Configuration Files](/cloud-docs/api-docs/configuration-versions#download-configuration-files) endpoints.

### 2022-03-11

* Introduced [Archiving Configuration Versions](/cloud-docs/workspaces/configurations#archiving-configuration-versions).
  * Updated [Configuration Versions](/cloud-docs/api-docs/configuration-versions#attributes) to add new `fetching` and `archived` states.
  * Updated [Runs](/cloud-docs/api-docs/run#attributes) to add new `fetching` state.
  * Added the [Archive a Configuration Version](/cloud-docs/api-docs/configuration-versions#archive-a-configuration-version) endpoint.

### 2022-02-25

* Updated [Workspace Run Tasks](/cloud-docs/api-docs/run-tasks#show-a-run-task) to add new `enabled`attribute.

### 2021-12-09

* Added `variables` field for POST /runs and the run resource, enabling run-specific variable values.

### 2021-12-03

* OAuth API updated to handle `secret` and `rsa_public_key` fields for POST/PUT.

### 2021-11-17

* Added pagination support to the following endpoints:
  * [Feature Sets](/cloud-docs/api-docs/feature-sets#list-feature-sets) - `GET /feature-sets`
  * [Notification Configurations](/cloud-docs/api-docs/notification-configurations#list-notification-configurations) - `GET /workspaces/:workspace_id/notification-configurations`
  * [Oauth Clients](/cloud-docs/api-docs/oauth-clients#list-oauth-clients) - `GET /organizations/:organization_name/oauth-clients`
  * [Oauth Tokens](/cloud-docs/api-docs/oauth-tokens#list-oauth-tokens) - `GET /oauth-clients/:oauth_client_id/oauth-tokens`
  * [Organization Feature Sets](/cloud-docs/api-docs/feature-sets#list-feature-sets-for-organization) - `GET /organizations/:organization_name/feature-sets`
  * [Organizations](/cloud-docs/api-docs/organizations#list-organizations) - `GET /organizations`
  * [Policy Checks](/cloud-docs/api-docs/policy-checks#list-policy-checks) - `GET /runs/:run_id/policy-checks`
  * [Policy Set Parameters](/cloud-docs/api-docs/policy-set-params#list-parameters) - `GET /policy-sets/:policy_set_id/parameters`
  * [SSH Keys](/cloud-docs/api-docs/ssh-keys#list-ssh-keys) - `GET /organizations/:organization_name/ssh-keys`
  * [User Tokens](/cloud-docs/api-docs/user-tokens#list-user-tokens) - `GET /users/:user_id/authentication-tokens`

### 2021-11-11

* Introduced the [Variable Sets](/cloud-docs/api-docs/variable-sets) endpoints for viewing and administering Variable Sets

### 2021-11-18

* Introduced the [Registry Providers](/cloud-docs/api-docs/providers) endpoint to manage public providers for a
  private registry. These endpoints will be available in the folowing TFE Release: `v202112-1`

### 2021-08-18

* Introduced the [State Version Outputs](/cloud-docs/api-docs/state-versions) endpoint to retrieve the Outputs for a
  given State Version

### 2021-08-11

* **BREAKING CHANGE:** Security fix to [Configuration versions](/cloud-docs/api-docs/configuration-versions): upload-url attribute for [uploading configuration files](/cloud-docs/api-docs/configuration-versions#upload-configuration-files) is now only available on the create response.

### 2021-07-30

* Introduced Workspace Tagging
  * Updated [Workspaces](/cloud-docs/api-docs/workspaces):
    * added `tag-names` attribute.
    * added `POST /workspaces/:workspace_id/relationships/tags`
    * added `DELETE /workspaces/:workspace_id/relationships/tags`
  * Added [Organization Tags](/cloud-docs/api-docs/organization-tags).
  * Added `tags` attribute to [`tfrun`](/cloud-docs/sentinel/import/tfrun)

### 2021-07-19

* [Notification configurations](/cloud-docs/api-docs/notification-configurations): Gave organization tokens permission to create and manage notification configurations.

### 2021-07-09

* [State versions](/cloud-docs/api-docs/state-versions): Fixed the ID format for the workspace relationship of a state version. Previously, the reported ID was unusable due to a bug.
* [Workspaces](/cloud-docs/api-docs/workspaces): Added `locked_by` as an includable related resource.

### 2021-06-8

* Updated [Registry Module APIs](/cloud-docs/api-docs/modules).
  * added `registry_name` scoped APIs.
  * added `organization_name` scoped APIs.
  * added [Module List API](/cloud-docs/api-docs/modules#list-registry-modules-for-an-organization).
  * updated [Module Delete APIs](/cloud-docs/api-docs/modules#delete-a-module) (see deprecation note below).
  * **CLOUD**: added public registry module related APIs.
* **DEPRECATION**: The following [Registry Module APIs](/cloud-docs/api-docs/modules) have been replaced with newer apis and will be removed in the future.
  * The following endpoints to delete modules are replaced with [Module Delete APIs](/cloud-docs/api-docs/modules#delete-a-module)
    * `POST /registry-modules/actions/delete/:organization_name/:name/:provider/:version` replaced with `DELETE /organizations/:organization_name/registry-modules/:registry_name/:namespace/:name/:provider/:version`
    * `POST /registry-modules/actions/delete/:organization_name/:name/:provider` replaced with `DELETE /organizations/:organization_name/registry-modules/:registry_name/:namespace/:name/:provider`
    * `POST /registry-modules/actions/delete/:organization_name/:name` replaced with `DELETE /organizations/:organization_name/registry-modules/:registry_name/:namespace/:name`
  * `POST /registry-modules` replaced with [`POST /organizations/:organization_name/registry-modules/vcs`](/cloud-docs/api-docs/modules#publish-a-private-module-from-a-vcs)
  * `POST /registry-modules/:organization_name/:name/:provider/versions` replaced with [`POST /organizations/:organization_name/registry-modules/:registry_name/:namespace/:name/:provider/versions`](/cloud-docs/api-docs/modules#create-a-module-version)
  * `GET /registry-modules/show/:organization_name/:name/:provider` replaced with [`GET /organizations/:organization_name/registry-modules/:registry_name/:namespace/:name/:provider`](/cloud-docs/api-docs/modules#get-a-module)

### 2021-05-27

* **CLOUD**: [Agents](/cloud-docs/api-docs/agents): added [delete endpoint](/cloud-docs/api-docs/agents#delete-an-agent).

### 2021-05-19

* [Runs](/cloud-docs/api-docs/run): added `refresh`, `refresh-only`, and `replace-addrs` attributes.

### 2021-04-16

* Introduced [Controlled Remote State Access](https://www.hashicorp.com/blog/announcing-controlled-remote-state-access-for-terraform-cloud-and-enterprise).

  * [Workspaces](/cloud-docs/api-docs/workspaces):

### 2021-04-13

### 2021-03-23

* **ENTERPRISE**: `v202103-1` Introduced [Share Modules Across Organizations with Terraform Enterprise](https://www.hashicorp.com/blog/share-modules-across-organizations-terraform-enterprise).
  * [Admin Organizations](/cloud-docs/api-docs/admin/organizations):
    * added new query parameters to [List all Organizations endpoint](/cloud-docs/api-docs/admin/organizations#query-parameters)
    * added module-consumers link in `relationships` response
    * added [update module consumers endpoint](/cloud-docs/api-docs/admin/organizations#update-an-organization-39-s-module-consumers)
    * added [list module consumers endpoint](/cloud-docs/api-docs/admin/organizations#list-module-consumers-for-an-organization)
  * [Organizations](/cloud-docs/api-docs/organizations): added [Module Producers](/cloud-docs/api-docs/organizations#show-module-producers)
  * **DEPRECATION**: [Admin Module Sharing](/cloud-docs/api-docs/admin/module-sharing): is replaced with a new JSON::Api compliant [endpoint](/cloud-docs/api-docs/admin/organizations#update-an-organization-39-s-module-consumers)

### 2021-03-18

* **CLOUD**: Introduced [New Workspace Overview for Terraform Cloud](https://www.hashicorp.com/blog/new-workspace-overview-for-terraform-cloud).

### 2021-03-11

* Added [VCS Events](/cloud-docs/api-docs/vcs-events), limited to GitLab.com connections.

### 2021-03-08

* [Workspaces](/cloud-docs/api-docs/workspaces): added `current_configuration_version` and `current_configuration_version.ingress_attributes` as includable related resources.
